On évoque souvent la puissance du Framework Rails grâce à tous les Helpers disponibles et le degré de customisation possible.
Mais on oublie régulièrement de vanter les mérites d'Rspec.

Tout le monde en a déjà entendu parler mais il est important de rendre à César ce qui lui est dû.

Par default, Rspec n'est pas installé sur Rails.
Le framework de test retenu par la Core Team Rails est Minitest.

Curieusement il y a quand même un consensus de la communauté Ruby sur l'utilisation d'Rspec.
Vous devrez donc l'installer vous-même sur vos projets si vous souhaitez l'utiliser, heureusement c'est assez simple et la Gem est bien construite.


La difference entre Minitest et Rspec réside dans la syntaxe et les librairies complémentaires disponibles.
Rspec est plébicité dans de nombreux projets Rails, mais vous trouverez parfois des équipes qui ont préféré Minitest.

Pour vous faire une idée plus précise le mieux est d'essayer les deux.
Personnellement j'utilise Rspec par default sur tous mes projets, y compris les projets Ruby Non Rails.

Pour l'installation, vous pouvez vous référer à la Documentation sur Github directement:
-> https://github.com/rspec/rspec-rails (RAILS)
-> https://github.com/rspec/rspec-metagem (NON RAILS)

Une fois installé, le fonctionnement est assez simple.

  • Pour lancer tous les specs: 
  $ rspec
  • Pour lancer un fichier test spécifique: 
  $ rspec spec/models/my_model_spec.rb
  • Pour lancer la ligne 33 d'un fichier test: 
  $ rspec spec/models/my_model_spec.rb:33

Pour plus d'informations concernant les bonnes pratiques de test, vous pouvez vous référer à la Documentation sur les tests (link).
